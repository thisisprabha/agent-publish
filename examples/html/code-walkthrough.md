# How Fingerprint Caching Works

## Overview

The `GitPublisher` avoids duplicate publishes by computing a SHA256 fingerprint of the rendered HTML.

```python
from pathlib import Path
import hashlib

def fingerprint(path: Path) -> str:
    content = path.read_text(encoding="utf-8")
    return hashlib.sha256(content.encode()).hexdigest()[:12]

# Usage
cache_file = repo / ".agent_publish_cache"
if cache_file.exists():
    known = set(cache_file.read_text().splitlines())
    if fingerprint(html_path) in known:
        return "skipped (duplicate)"
```

## Edge Cases

- Whitespace-only changes are still caught (SHA256 is byte-exact)
- Deleting an entry and republishing the same content is treated as new
- Cache is stored in `repo/.agent_publish_cache`
