# AP-038: Interactive Post Preview

**Status:** Completed
**Completed:** 2026-06-08

## What was built

A `preview` subcommand that spins up an ephemeral HTTP server for draft HTML:
- **Port**: 8765 (default, configurable via `--port`)
- **Temp directory**: Created via `tempfile.mkdtemp()`, auto-deleted on shutdown
- **Noindex**: Auto-injects `<meta name="robots" content="noindex, nofollow">` to prevent search indexing
- **LiveReload**: WebSocket-like notification server on port+1 (8766). Client-side script reconnects and reloads on change
- **File watcher**: watchdog-based observer on the input file's parent directory. Debounced 500ms rebuilds
- **SIGINT handler**: Ctrl+C gracefully stops server, observer, and cleans up temp files
- **Feature parity**: Supports all existing CLI flags — `--theme`, `--skill`, `--humanize`, `--tldr`, `--tags`, `--no-toc`, `--no-mermaid`, `--favicon`, `--author`, `--site_title`, `--template`, `--custom-css`, `--og-image`

## CLI usage

```bash
agent-publish preview report.md
agent-publish preview report.md --port 8888 --theme minimal --tldr --tags
agent-publish preview slides.md --skill deck --port 9000
```

## Architecture

```
PreviewServer
├── _build()          → convert_file() + noindex meta + LiveReload script
├── _serve()          → http.server in background thread
├── _setup_file_watcher() → watchdog Observer on input file parent
├── _setup_websocket_server() → raw socket server on port+1
├── _notify_reload()  → connect to ws server, send "reload"
├── _setup_signal_handlers() → SIGINT → shutdown event
└── start()           → orchestrate all components, cleanup on exit
```
