## What was built

A `skills/` directory system where each output type is a folder containing:
- `SKILL.md` — lightweight spec with title, description, template_vars
- `template.html` — custom HTML template with standard placeholders
- `assets/` (optional) — images, fonts, or other assets to copy to output

## Built-in skills (4)

| Skill | Purpose | Visual treatment |
|---|---|---|
| `article` | Long-form content | Centered, editorial, comfortable line-height |
| `briefing` | Daily/compact updates | High-density, tighter margins |
| `changelog` | Release notes | Dated list, version-oriented |
| `deck` | Presentation-style | Wide layout, larger typography |

## CLI usage

```bash
agent-publish publish report.md --skill briefing
agent-publish publish slides.md --skill deck --watch
```

## How skill loading works

1. `discover_skills(base_path)` scans for subdirectories with `SKILL.md` + `template.html`
2. `load_skill(skill_path)` reads the SKILL.md metadata and template content
3. CLI `--skill` flag triggers skill discovery and loads the matching skill
4. Skill template takes precedence over both `DEFAULT_TEMPLATE` and `--template` override
5. Skill assets are copied to output dir and tracked in `assets_copied`
